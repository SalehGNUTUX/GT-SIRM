## Gotchas (do not rediscover)

### Linux Electron+GTK ignores `defaultPath` in `dialog.showOpenDialog`
Confirmed bug on KDE/Plasma + GNOME. Bypass via kdialog (KDE) / zenity (GNOME). Sentinel `_DLG_UNAVAILABLE` distinguishes "tool failed" from "user cancelled" — cancellation MUST short-circuit (else user sees N dialogs).

### File input interceptor (v0.12.11)
Replaces Chromium picker with native dialog opened in correct workdir subfolder. Listener on BOTH `input` (capture) + `.fu-area` parent. `pointer-events:none` + `_workdirInstalled` + `_bypassWorkdir` flags.

### WebM/Opus duration is Infinity from MediaRecorder
MediaRecorder produces blobs without duration metadata. Audio element gets `duration === Infinity`. **Solution**: seek trick — set `currentTime = 1e101` then back to 0; this forces Chromium to scan the container. Applied in `onBgAudio` for ALL custom audio uploads.

### Autosave clearing dirty flag (v0.13.0 fix)
When autosave writes to localStorage (no .gtsirm path yet), it MUST NOT clear `S.projectDirty` — otherwise close warning is suppressed. Only `saveProjectToPath()` to a real .gtsirm file clears dirty.

### Collapsible sections: pre-existing HTML imbalance in tab-rec
Original HTML had a missing `</div>` for per-slice section. With `<div>` invisible, with `<details>` it folds recvid + Quran timing too. Fix: explicit `</details>` after `free-per-slice-stats`.

### Regex `\b` after "sec" matches "sec-t"
When converting `<div class="sec">` to `<details>`, regex `class="sec\b[^"]*"` ALSO matches `class="sec-t"`. **Use negative lookahead**: `class="sec(?!-)[^"]*"`.

### Android Capacitor 6 + classic scripts
`window.Capacitor.Plugins.App` and `.Filesystem` are auto-injected by the native bridge when the WebView loads. No ESM imports needed. Code MUST gracefully handle `!Capacitor` (web/desktop) AND `!Capacitor.Plugins.App` (plugin not bundled) without crashing.

### Android icons get overwritten by `cap sync`
Capacitor regenerates default green icons in mipmap-* during `cap sync`. **Solution**: `build-apk.sh` runs `gen-android-icons.py` AFTER sync. The script generates 3 files per density (legacy square + round + adaptive foreground) and updates `ic_launcher_background.xml` to `#040c08`.

### Android intent-filter pathPattern
For `.gtsirm` to appear in "Open with" on Android, intent-filter needs multiple `pathPattern` entries (`.*\\.gtsirm`, `.*\\..*\\.gtsirm`, etc.) because Android matches greedily. Also: separate intent-filters for `file://`/`content://` schemes AND for `application/json` mimeType.

### V2 export and HTMLVideoElement (recvid)
V2 deterministic export iterates frames at fixed timestamps. HTMLVideoElement needs explicit `currentTime = t` + wait for `seeked` event via `seekVideoToTime(S.recVidEl, t)` (800ms timeout).

### Beforeunload + file uploads
`beforeunload` only fires if `S.projectDirty=true`. Dirty-tracker hooks DOM inputs but skips `type=file`. Call `markProjectDirty()` explicitly in `onBgMedia`, `onBgAudio`, `addBgVidItem`, `onRecVidFile`.

### Custom close dialog (Electron)
`beforeunload` in Electron renderer cannot show custom dialogs. Pattern: main.js intercepts close → IPC `request-close-confirm` → renderer shows 3-button modal → IPC `confirm-close` releases.

### YCbCr distance can't distinguish black from white
Both `(0,0,0)` and `(255,255,255)` have Cb=128, Cr=128. Plain YCbCr chromakey fails for grayscale. Detect saturation `(max-min)<25` → luminance-mode (max for dark, min for light).

### Asset serialization needs S.bgImgFile / S.bgAudioFile / S.recVidFile
File handlers MUST store original `File` on S for project save: `S.bgImgFile = file; S.bgAudioFile = file; S.recVidFile = file;` Original handler only saved derived objects (Image/Audio).

### `S.freePerSlice` reuse trap (v0.8.6)
Per-slice section reads from `S.freePerSlice` when rendering. If user applies Hadith → Free Text → clicks 🎚️, OLD slices render. `openPerSliceSmart()` sets `S.freePerSlice = null` BEFORE `buildPerSliceList()`. Also opens containing `<details class="sec">` (v0.13.0).

### Audio duration: priority of sources (v0.8.5)
`getActiveAudioDuration()` checks **recvid → trim → bgAudio**. Insert new audio sources here AND update `syncVersesToActiveAudio` to use it.

### RPM build picks stale DEB (alien fallback)
`alien -r -g <newest-deb>`. If older versioned DEB is in `dist/`, alien may pick wrong. **Always delete old version artifacts before building.**

### Hotfix without bumping version: replace assets on the same release

### Adding a new content module (v0.10 pattern, 9 steps)
1. **Data file** `X-data.js` (classic script): `window.X_DATA = { categories: [{id, name, icon, items: [{n, text, source, ...}]}] }`.
2. **`<script src="X-data.js">`** in `index.html` after `hadith-data.js` (both versions).
3. **HTML section** `<details class="sec" data-module="X">` in `tab-rec` after azkar (category select + search + select size=6 + preview + apply + per-slice).
4. **Settings toggle** `<input id="mod-X">` + entry in `MODULES` with `impl: true`.
5. **JS**: `initXModule()` populates UI + listeners. `applyX(item, cat)` → `clearOtherSourcesUI()` → `splitArabicTextSmart` → `calcEffectiveSliceDuration` → fill `S.verses` + `S.freePerSlice`.
6. **Call `initXModule()`** in `DOMContentLoaded` sequence after `initAzkarModule()`.
7. **Service Worker** adds `./X-data.js` to `OPTIONAL` + cache version bump.
8. **`build-apk.sh`** copies `X-data.js` to `www/`.
9. **Docs**: README + ROADMAP + CHANGELOG.

### Module mutual exclusion (v0.8.14 + v0.10.0)
`MODULES` with `enforceSingleModuleActive(state)` ensures only one content module active. At start of every `applyX`, call `clearOtherSourcesUI()`. Free text is NOT a module since v0.8.16 — permanent tool, but auto-disabled when applying content module.
