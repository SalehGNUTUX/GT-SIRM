# خارطة طريق GT-SIRM

> آخر تحديث: 2026-06-06 — بدء البنية الأساسيّة (v0.1.0)
> هذا الملف مشترك بين نسختَي سطح المكتب والويب.

---

## 🌙 إعلان النيّة

بُنِيَ هذا المشروع ابتغاءَ وجه الله الكريم، خِدمةً للإسلام والمسلمين. كلّ ميزة تُضاف تخدم هذا الغرض، ولا تُقبل ميزة تخالف الشرع أو تفتح باباً لمعصية.

---

## 🎯 المبادئ الموجِّهة

1. **تماثل المزايا بين النسختَين** — كلّ ميزة تُطبَّق في `GT-SIRM-DESKTOP/` و `GT-SIRM-WEB/` في نفس Commit
2. **توارُث ذكيّ** — البناء على ميزات GT-SQRM v3.0 المُختبَرة، لا إعادة كتابة
3. **العمل دون اتصال** — كلّ المحتوى الإسلاميّ متاح offline (قرآن، حديث، أذكار، أدعية)
4. **مرونة الاستيراد** — المستخدم يُحضِر نصّه أو صوته أو فيديوه بأبسط طريقة
5. **توگل الوحدات** — كلّ وحدة محتوى قابلة للإلغاء، لا فرض على المستخدم
6. **التزام شرعيّ** — التحقّق من تصحيح الأحاديث، توثيق مصادر الأقوال

## 📚 خُطّة تَوسيع وَحدة الحديث الشريف

### الحالة الحاليّة (v1.0.0)
- **90 حَديثاً** فقط: الأربعون النووية (40) + مُختارات من رياض الصالحين (50)
- مَصدر داخليّ: `hadith-data.js` (classic script يَضَع `window.HADITH_DATA`)

### المَصدر المُقترَح لِلتَوسعة
**[AhmedBaset/hadith-json](https://github.com/AhmedBaset/hadith-json)** — مُستودع مَفتوح يَحوي **50,884 حَديثاً** في **17 مَجموعة** بصيغة JSON.

#### المَجموعات المُتاحة
**الكُتُب التِّسعة:**
1. صَحيح البُخاريّ
2. صَحيح مُسلم
3. سُنَن أبي داود
4. جامِع التِّرمذيّ
5. سُنَن النَّسائيّ
6. سُنَن ابن ماجَه
7. مُوطَّأ مالك
8. مُسنَد أحمد *(فُصول 8-30 ناقصة في المَصدر)*
9. سُنَن الدَّارميّ

**مَجموعات إضافيّة:**
10. رياض الصالحين *(مَوجود جُزئيّاً في v1.0)*
11. الشَّمائل المُحَمَّديّة
12. بُلوغ المَرام
13. الأدب المُفرَد
14. مِشكاة المَصابيح
15. الأربعون النَّوويّة *(مَوجودة في v1.0)*
16. الأربعون القُدسيّة
17. أربعون الشاه ولي الله

### بِنية البيانات (Schema)
```json
{
  "id": 1,
  "chapterId": 1,
  "bookId": 1,
  "arabic": "...",
  "english": { "narrator": "...", "text": "..." }
}
```

### خُطّة التَنفيذ التَدريجيّ
- **v1.1 — الكُتُب السِتّة (الصِحاح + السُنَن الأربعة)** ≈ 35,000 حَديث
  - تَنزيل JSON من الـrelease tag (pin to specific version لتَجنّب breaking changes)
  - تَحويل لصيغة `HADITH_DATA` الحاليّة (chapters / hadiths)
  - **إضافة تَصحيح الإسناد** يَدويّاً أو من مَصدر مُكمِّل (الـjson لا يَحوي grading) — مَطلوب للإلتزام بقَواعد المَشروع
  - تَجزئة بحَسب الكِتاب (lazy-load) لتَجنّب bundle ضَخم
- **v1.2 — البَقيّة:** مُوطَّأ مالك + مُسنَد أحمد + سُنَن الدَّارميّ + المَجموعات الإضافيّة
- **بحث + تَصفية** بالكِتاب/الباب/الراوي/درجة التَصحيح

### اِعتبارات شَرعيّة وفِكريّة
- **شارة التَصحيح إلزاميّة** لكلّ حَديث (صَحيح/حَسَن/ضَعيف). إن لم تَتوفّر في المَصدر، إمّا يَدويّاً أو من مَصدر آخَر مَوثوق (مثل توثيقات الألبانيّ أو شُعَيب الأَرنؤوط).
- **التَّحقُّق من النَصّ العَربيّ** قبل الإدراج — لا اعتماد على scrapers بشَكل أعمى.
- **تَقسيم الـbundle** لتَجنّب تَحميل 50MB JSON على الهاتف.

### اعتبارات تِقنيّة
- المَلفّ الواحد للبخاريّ ~7,000 حَديث = ~5MB JSON. مَجموع الـ17 = ~35MB.
- **Lazy-load** عَبر `<script>` ديناميكيّ عند فَتح الوَحدة لكلّ كِتاب.
- **Service Worker** يُخزّن المَجموعة عند أوّل استخدام.
- **Pin to tag** لتَجنّب breaking changes (مَوصى به في المُستودع المَصدر).
- **Attribution** في تَبويب "حول": "بيانات الحَديث: AhmedBaset/hadith-json (مَصدرها sunnah.com)"

## 🔗 العلاقة مع GT-SQRM

GT-SIRM **يَرِث** كلّ ميزات GT-SQRM v3.0 — لا يُلغيها. النسختان تتعايشان:
- **GT-SQRM** = صانع ريلز مُكرَّس للقرآن — بسيط ومركَّز
- **GT-SIRM** = صانع ريلز إسلاميّ شامل — مرن وموسَّع
- **معالج "🔄 استيراد إعدادات GT-SQRM"** داخل GT-SIRM لنقل: القوالب · القرّاء · الخطوط
